# Cloud Deployment Guide (Render / Railway)

This guide provides instructions to deploy the Billing Software (Spring Boot backend + PostgreSQL database + React/Vite frontend) to cloud platforms like **Render** or **Railway**.

---

## 1. Database Setup

First, create a PostgreSQL database instance on your cloud provider.

### Option A: Railway (Recommended)
1. Log in to Railway and click **New Project** -> **Provision PostgreSQL**.
2. Once provisioned, click on the **PostgreSQL** service and navigate to the **Variables** tab to copy the database connection details.

### Option B: Render
1. Log in to Render and click **New** -> **PostgreSQL**.
2. Choose a name and database name. Select the free tier.
3. Once created, copy the **Internal Database URL** (for backend services deployed on Render) or **External Database URL**.

---

## 2. Backend Deployment (Spring Boot)

Deploy the backend as a container web service using the provided Dockerfile.

### Option A: Railway
1. Click **New** -> **GitHub Repo** and select this repository.
2. Go to the service's **Settings** tab on the Railway dashboard, find the **Root Directory** field, and set it to `backend`.
3. Under the **Variables** tab, click **New Variable** and configure them using Railway's dynamic service referencing (which automatically syncs the credentials from your PostgreSQL service):
   * `SPRING_DATASOURCE_URL`: `jdbc:postgresql://${{Postgres.PGHOST}}:${{Postgres.PGPORT}}/${{Postgres.PGDATABASE}}`
   * `SPRING_DATASOURCE_USERNAME`: `${{Postgres.PGUSER}}`
   * `SPRING_DATASOURCE_PASSWORD`: `${{Postgres.PGPASSWORD}}`
   * `APP_CORS_ALLOWED_ORIGINS`: `https://your-deployed-frontend-url.railway.app`
   * `APP_JWT_SECRET`: A secure, randomly generated 256-bit string.
   * `PORT`: `8080`
   *(Note: Make sure to replace `Postgres` with the exact name of your PostgreSQL service if it is named differently in your Railway project, e.g., `postgres` or `database`)*
4. Deploy the service.

### Option B: Render
1. Click **New** -> **Web Service**.
2. Connect your GitHub repository.
3. Set the following details:
   * **Name**: `billing-backend`
   * **Root Directory**: `backend`
   * **Runtime**: `Docker`
4. Under **Advanced** -> **Add Environment Variable**:
   * `SPRING_DATASOURCE_URL`: `jdbc:postgresql://<DB_HOST>:<DB_PORT>/<DB_NAME>` (Use the Internal Database URL from Step 1, prepended with `jdbc:`)
   * `SPRING_DATASOURCE_USERNAME`: `<DB_USER>`
   * `SPRING_DATASOURCE_PASSWORD`: `<DB_PASSWORD>`
   * `APP_CORS_ALLOWED_ORIGINS`: `https://your-deployed-frontend-url.onrender.com`
   * `APP_JWT_SECRET`: A secure, randomly generated 256-bit string.
5. Click **Create Web Service**.

---

## 3. Frontend Deployment (React / Vite Static Site)

Deploy the frontend as a static site (connected to your Git repository).

### Option A: Render (Free Static Site Hosting)
1. Click **New** -> **Static Site**.
2. Connect your GitHub repository.
3. Set the following details:
   * **Name**: `billing-frontend`
   * **Root Directory**: `frontend`
   * **Build Command**: `npm run build`
   * **Publish Directory**: `dist`
4. Under **Advanced** -> **Add Environment Variable**:
   * `VITE_API_URL`: `https://your-deployed-backend-url.onrender.com` (use the URL generated by Render for your backend service)
5. Click **Create Static Site**.

### Option B: Railway
1. Click **New** -> **GitHub Repo** and select this repository.
2. In settings, set the **Root Directory** to `frontend`.
3. Under **Variables**, add:
   * `VITE_API_URL`: `https://your-deployed-backend-url.railway.app`
4. Deploy the service.

---

## 🔒 Security Best Practices
Be sure to override the JWT secret key in your production environment by adding the following variable to your deployed backend service:
* `APP_JWT_SECRET`: A secure, randomly generated 256-bit string.

Also set the CORS allowed origins to your deployed frontend URL:
* `APP_CORS_ALLOWED_ORIGINS`: `https://your-deployed-frontend-url`

Do not include a trailing slash in the frontend URL. For multiple frontend URLs, separate them with commas.
